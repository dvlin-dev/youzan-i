# 交互与体验原则

> 规范事实源。界面硬要求是"有逻辑、有美感"。本文定义"有逻辑"的判据与关键交互模式。

## 设计立场：这是操作台，不是落地页

第一性原理：它是仓管、采购每天盯 8 小时的**高频操作台**，不是给人看三秒的落地页。由此取舍：

- **要信息密度，不要留白氛围感**：紧凑表格 + 固定表头 + 冻结首列（款号），而非卡片瀑布流。
- **要效率，不要"好看但慢"**：批发工作流是**按矩阵走、扫码走、批量走**——一个款号一次入 5 色 × 6 码 = 30 个 SKU，界面必须贴着这个工作流长。
- **"有逻辑"的判据**：每个页面都为某角色的某高频决策服务，信息密度对齐该决策，且空态 / 载入态 / 异常态都被设计过（"今天没有低库存"和"接口挂了"是两件事）。

## 角色化首屏

登录即落到该角色的高频任务：仓管 → 库存看板 + 待复核；采购 → 低库存 + 在途采购 + 对账；老板 → 全局 KPI + 盘点差异。仓管首屏不出现金额。

## 关键交互模式

| 模式 | 治的痛点 | 怎么做 |
| --- | --- | --- |
| **SKU 矩阵录入** | 一款几十个 SKU，点 30 次"新增"没人受得了 | 行 = 颜色、列 = 尺码的矩阵，Tab / 方向键穿梭，像 Excel 一样快；扫码为主输入 |
| **命令面板 ⌘K** | 高频操作藏在多级菜单里 | 一处搜索页面 / SKU / 指令，键盘直达 |
| **乐观更新 + 撤销** | 等接口转圈、误操作无法挽回 | 立即反馈 + 5 秒 Undo toast；写操作仍进待复核 |
| **三态设计** | 空白页分不清"没数据"还是"挂了" | 空态（含正向空态"库存健康"）/ 载入骨架 / 错误态分别设计 |
| **点任意数字 → 流水链** | 库存对不上时没人能解释"这个 12 怎么来的" | 任意库存 / 差异数字可点，展开该 SKU 全部流水（带单号、经手人、前后结存）——"库存 = 流水累加"的可视化兑现 |
| **AI 助手侧栏** | 不会用复杂表单 | 自然语言连续出入库 / 对账（agent 一轮多步），直接生成待复核单、收尾汇报并提示去审批 |

## 从 Excel 平滑迁移（同理心）

- 支持粘贴 Excel 区域批量录入；保留"款号 / 颜色 / 尺码"这套用户已有的心智词汇。
- 危险操作（删单、改成本价、整单作废）二次确认 + 待复核审批 + 留痕。
- 移动 / 现场适配：大按钮（≥44pt）、单手拇指可达、扫码优先、断网进本地队列待同步。

## 视觉与无障碍

- 颜色语义统一：红 = 缺货 / 断码、橙 = 偏低、绿 = 健康；颜色不单独承载信息，配图标 + 文案。
- 对比度 ≥ WCAG AA（正文 4.5:1）、全功能可纯键盘到达、焦点环可见。
- 设计 token 集中维护（配色 / 间距 / 圆角 / 阴影），组件消费 token，不散落硬编码颜色。原型采用"暖色账册操作台"风格（petrol-teal 主色 + bone 背景），刻意避开通用 AI 审美（Inter + 紫色渐变）。
